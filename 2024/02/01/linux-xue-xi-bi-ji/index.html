Linux学习笔记

文件结构

  • /bin：
    bin 是 Binaries (二进制文件) 的缩写, 这个目录存放着最经常使用的命令。

  • /boot：
    这里存放的是启动 Linux 时使用的一些核心文件，包括一些连接文件以及镜像文件。

  • /dev ：
    dev 是 Device(设备) 的缩写, 该目录下存放的是 Linux 的外部设备，在 Linux 中访问设备的方式和访问文件的方式是相同的。

  • /etc：
    etc 是 Etcetera(等等) 的缩写,这个目录用来存放所有的系统管理所需要的配置文件和子目录。

  • /home：
    用户的主目录，在 Linux 中，每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的，如上图中的 alice、bob 和 eve。

  • /lib：
    lib 是 Library(库) 的缩写这个目录里存放着系统最基本的动态连接共享库，其作用类似于 Windows 里的 DLL 文件。几乎所有的应用程序都需要用到这些共享库。

  • /lost+found：
    这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。

  • /media：
    linux 系统会自动识别一些设备，例如U盘、光驱等等，当识别后，Linux 会把识别的设备挂载到这个目录下。

  • /mnt：
    系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将光驱挂载在 /mnt/ 上，然后进入该目录就可以查看光驱里的内容了。

  • /opt：
    opt 是 optional(可选) 的缩写，这是给主机额外安装软件所摆放的目录。比如你安装一个ORACLE数据库则就可以放到这个目录下。默认是空的。

  • /proc：
    proc 是 Processes(进程) 的缩写，/proc 是一种伪文件系统（也即虚拟文件系统），存储的是当前内核运行状态的一系列特殊文件，这个目录是一个虚拟的目录，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。
    这个目录的内容不在硬盘上而是在内存里，我们也可以直接修改里面的某些文件，比如可以通过下面的命令来屏蔽主机的ping命令，使别人无法ping你的机器：

    echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_all

  • /root：
    该目录为系统管理员，也称作超级权限者的用户主目录。

  • /sbin：
    s 就是 Super User 的意思，是 Superuser Binaries (超级用户的二进制文件) 的缩写，这里存放的是系统管理员使用的系统管理程序。

  • /selinux：
    这个目录是 Redhat/CentOS 所特有的目录，Selinux 是一个安全机制，类似于 windows 的防火墙，但是这套机制比较复杂，这个目录就是存放selinux相关的文件的。

  • /srv：
    该目录存放一些服务启动之后需要提取的数据。

  • /sys：

    这是 Linux2.6 内核的一个很大的变化。该目录下安装了 2.6 内核中新出现的一个文件系统 sysfs 。

    sysfs 文件系统集成了下面3种文件系统的信息：针对进程信息的 proc 文件系统、针对设备的 devfs 文件系统以及针对伪终端的 devpts 文件系统。

    该文件系统是内核设备树的一个直观反映。

    当一个内核对象被创建的时候，对应的文件和目录也在内核对象子系统中被创建。

  • /tmp：
    tmp 是 temporary(临时) 的缩写这个目录是用来存放一些临时文件的。

  • /usr：
    usr 是 unix shared resources(共享资源) 的缩写，这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似于 windows 下的 program files 目录。

  • /usr/bin：
    系统用户使用的应用程序。

  • /usr/sbin：
    超级用户使用的比较高级的管理程序和系统守护程序。

  • /usr/src：
    内核源代码默认的放置目录。

  • /var：
    var 是 variable(变量) 的缩写，这个目录中存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件。

  • /run：
    是一个临时文件系统，存储系统启动以来的信息。当系统重启时，这个目录下的文件应该被删掉或清除。如果你的系统上有 /var/run 目录，应该让它指向 run。

常用命令

  1. ls (list)

    • 作用：列出目录内容。
    • 常用参数：
      • -l：以长格式显示详细信息。
      • -a：显示所有文件，包括以.开头的隐藏文件。

  2. cd (change directory)

    • 作用：改变当前工作目录。
    • 参数：目标目录的路径。

  3. cp (copy)

    • 作用：复制文件或目录。
    • 常用参数：
      • -r：递归地复制目录及其内容。
      • -i：在覆盖文件之前进行提示。

  4. mv (move)

    • 作用：移动文件或目录，也可用于重命名。
    • 常用参数：
      • -i：在覆盖文件之前进行提示。

  5. rm (remove)

    • 作用：删除文件或目录。
    • 常用参数：
      • -r：递归删除目录及其内容。
      • -f：强制执行，不进行提示。

  6. mkdir (make directory)

    • 作用：创建新目录。
    • 常用参数：
      • -p：递归创建目录，如果父目录不存在也会创建。

  7. rmdir (remove directory)

    • 作用：删除空目录。
    • 参数：目标目录的路径。

  8. touch

    • 作用：更新文件的访问和修改时间，也可用于创建新文件。
    • 参数：文件名。

  9. cat (concatenate)

    • 作用：连接文件并打印到标准输出。
    • 参数：文件名。

  10. grep (global regular expression print)

    • 作用：在文件中搜索文本模式。
    • 常用参数：
      • -i：忽略大小写。
      • -r：递归搜索目录及其子目录。

  11. find

    • 作用：在文件系统中查找文件。
    • 常用参数：
      • 路径：指定查找的起始路径。
      • -name：按文件名进行匹配。

  12. chmod (change mode)

    • 作用：更改文件权限。
    • 常用参数：
      • 使用数字表示权限，例如，chmod 755 filename。

  13. chown (change owner)

    • 作用：更改文件所有者。
    • 常用参数：
      • user:group：指定新的所有者和组。

  14. ps (process status)

    • 作用：显示当前进程的状态。
    • 常用参数：
      • aux：以详细信息的形式显示所有进程。

  15. 查看端口

    • netstat/ss -tulpn

      参数解释：

      • -t: 显示TCP连接
      • -u: 显示UDP连接
      • -l: 显示监听中的端口
      • -p: 显示进程标识（PID）
      • -n: 显示数值形式的地址和端口号

  16. kill

    • 作用：终止进程。
    • 参数：进程ID（PID）或进程名。

  17. 通用参数

    • -h 或 --help：显示帮助信息。
    • -v 或 --version：显示命令的版本信息。

  18. 文件/目录操作命令通用参数

    • -r 或 --recursive：递归操作，用于处理目录及其内容。
    • -f 或 --force：强制执行，不进行提示。

  19. 复制/移动命令通用参数

    • -i：在操作之前进行提示，询问用户是否确认执行。

  20. 目录创建命令通用参数

    • -p：递归创建目录，如果父目录不存在也会创建。

  21. 搜索命令通用参数

    • -i：忽略大小写。
    • -r 或 --recursive：递归搜索目录及其子目录。

  22. 权限相关命令通用参数

    • -v 或 --verbose：详细输出，显示更多信息。
    • -R 或 --recursive：递归地应用权限更改。

  23. 进程管理命令通用参数

    • -a：显示所有进程。
    • -u：显示与指定用户相关的进程。

硬链接和软链接（ln）

硬链接(不加参数):相当于将文件又创建了一份，删除其中一份不影响其他文件

软链接（加参数：-s）：相当于创建了一个快捷键

示例：ln -s 源目录/源文件 目标目录/目标文件

账户管理

  1. useradd

    • 作用：创建用户。
    • 常用参数：
      • -m：默认在当前目录自动创建用户。

  2. userdel

    • 作用：删除用户。
    • 常用参数：
      • -r：删除用户的文件。

  3. su、sudo

    • 作用：切换用户。
    • 说明：su必须告知密码，而sudo只是为账号临时分配一定操作权限。

  4. 修改用户名

    临时生效

    通过hostname进行修改，输入命令hostname 主机名，重新登录 shell 生效。这个命令的作用是暂时修改linux的主机名，它的存活时间是linux当前的运行时间，也就是在下一次重启前的运行时间内。一般修改以后立即生效，但这只是临时生效，不是永久生效，重启系统之后会还原。

    [root@centos7 ~]# hostname test
[root@centos7 ~]#

    重新登录shell之后主机名已经被修改

    [root@test ~]#

    永久方法

    通过修改配置文件/etc/sysconfig/network

    在这个配置文件中底部添加一条HOSTNAME=主机名(可自定义)，这个修改它能够实现永久修改linux的主机名，但是它不会立即生效，也不会重新登录shell生效，而是要重启系统才能生效。通过修改此配置文件，再配合hostname命令，可以实现立即永久修改linux的主机名，当然了，至少需要重启shell才可以。

    [root@test ~]# cat /etc/sysconfig/network
# Created by anaconda
NETWORKING_IPV6=no
PEERNTP=no
[root@test ~]#

    如果是 Ubuntu 系统，则需要修改文件 /etc/hostname， 将其对应的主机名修改为新的主机名。
    最后，需要将 /etc/hosts 中 127.0.0.1 对应的老主机名更换为新的主机名（如果有需要）。

    [root@test ~]# cat /etc/hosts
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
[root@test ~]#
    通过命令hostnamectl set-hostname 主机名修改

    centos7系统可以直接通过命令hostnamectl set-hostname 主机名进行修改，修改完毕后重新登录 shell 就可以了。

    [root@test ~]# hostnamectl set-hostname centos7
[root@test ~]#

进程管理

  1. 什么是进程

    • 每一个进程都会有一个父进程

    • 进程分两种形式：前台、后台

    • 一般服务器都是后台运行

  2. 命令

    • Ps -xx :查看当前运行的进程
      • ‘-a’ ：显示所有运行的进程
      • ‘-u’ ：以用户的信息显示进程
      • ‘-x’ ：显示后台运行进程的参数
    ps -aux | grep mysql #查看关于mysql的所有进程

firewall管理

  1. 查看防火墙状态：

    sudo firewall-cmd --state

  2. 启用防火墙：

    sudo systemctl start firewalld
或者
sudo service firewalld start

  3. 停止防火墙：

    sudo systemctl stop firewalld
或者
sudo service firewalld stop

  4. 重启防火墙：

    sudo systemctl restart firewalld
或者
sudo service firewalld restart

  5. 查看防火墙规则：

    #查看全部信息
sudo firewall-cmd --list-all
#只看端口
sudo firewall-cmd --list-ports

  6. 添加允许端口：

    #--permanent永久生效 --zone作用域
sudo firewall-cmd --zone=public --add-port=<端口号>/tcp --permanent

  7. 重新载入防火墙规则：

    sudo firewall-cmd --reload
或者
sudo systemctl restart firewalld.service

zsh安装

**Ubuntu/Kali/Debian 安装 **

# 安装zsh
apt install zsh

# 更改默认shell为zsh
chsh -s /bin/zsh

# 安装oh-my-zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

# 下载命令补全插件
git clone https://github.com/zsh-users/zsh-autosuggestions $ZSH_CUSTOM/plugins/zsh-autosuggestions

# zshrc配置文件中修改如下内容
plugins=(git zsh-autosuggestions)

# 重新加载zsh配置
zsh

yum install zsh

bash和zsh切换

  • 1.使用chsh命令：

    chsh命令允许你更改默认的shell。在终端中运行以下命令：

    chsh -s /bin/bash  # 切换到bash
chsh -s /bin/zsh  # 切换到zsh

这将分别将默认shell更改为bash或zsh。你需要重新登录才能看到更改。

  • 2.手动切换：

    如果你只是想在当前会话中切换shell，而不更改默认shell，你可以手动运行bash或zsh：

    bash  # 切换到bash
zsh   # 切换到zsh

vim使用

Vim 的基础使用方法：

1. 启动 Vim：

bashCopy code
vim

或者

bashCopy code
vim 文件名

2. Vim 模式：

  • Normal 模式：
    • 默认模式，用于导航、剪切、复制等。
  • Insert 模式：
    • 用于输入文本。
  • Visual 模式：
    • 用于选择文本块。

3. 基本移动：

  • h、j、k、l：
    • 分别是左、下、上、右，用于在 Normal 模式下移动光标。
  • w、b：
    • 分别是向前和向后移动一个单词。
  • gg、G：
    • 跳转到文件开头和文件末尾。

4. 编辑操作：

  • i、I、a、A：
    • 进入 Insert 模式，分别在当前光标前插入、行首插入、光标后插入、行尾插入。
  • o、O：
    • 在当前行的下方（o）或上方（O）插入新行。
  • x、dd：
    • 删除光标处的字符（x）或整行（dd）。

5. 复制和粘贴：

  • yy：
    • 复制当前行。
  • p：
    • 粘贴复制的内容。

6. 撤销和重做：

  • u：
    • 撤销上一步操作。
  • Ctrl + r：
    • 重做。

7. 搜索和替换：

  • /search：
    • 在 Normal 模式下输入 / 后跟要搜索的内容，按 Enter 开始搜索。
  • :s/old/new/g：
    • 替换文档中所有出现的 “old” 为 “new”。

8. 保存和退出：

  • :w：
    • 保存当前文件。
  • :q：
    • 退出 Vim。
  • :wq 或 :x：
    • 保存并退出。
  • :q!：
    • 强制退出，不保存更改。

9. Visual 模式：

  • v：
    • 逐字符选择。
  • V：
    • 逐行选择。
  • Ctrl + v：
    • 逐块选择（矩形选择）。

Vim 的高级使用方法：

1. 分屏和多窗口：

  • :split 或 :sp：
    • 水平分割窗口。
  • :vsplit 或 :vsp：
    • 垂直分割窗口。
  • Ctrl + w + 方向键：
    • 在窗口之间切换。

2. 标签页：

  • :tabnew 文件名：
    • 打开一个新标签页。
  • :tabnext 或 :tabn：
    • 切换到下一个标签页。
  • :tabprev 或 :tabp：
    • 切换到上一个标签页。
  • :tabclose 或 :tabc：
    • 关闭当前标签页。

3. 宏录制：

  • q + 字母：
    • 开始录制宏。例如，qa 开始录制到寄存器 a。
  • q：
    • 结束录制宏。
  • @ + 字母：
    • 执行宏。例如，@a 执行寄存器 a 中的宏。

4. 自定义配置：

  • ~/.vimrc 文件：
    • 你可以在此文件中添加 Vim 配置，定制你的编辑器环境。

5. 插件管理器：

  • 使用插件管理器如 Vundle、Pathogen 或 Vim-Plug，以便轻松管理和安装插件。

6. 语法高亮和主题：

  • 启用语法高亮：在 Normal 模式下输入 :syntax on。
  • 切换主题：在 Normal 模式下输入 :colorscheme 主题名称。
